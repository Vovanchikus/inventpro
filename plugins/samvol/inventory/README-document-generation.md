# Генерация документов по операциям

## Коротко

Модуль формирует DOCX по операции на основе выбранного шаблона и подставляет:

- данные операции и документов;
- данные из страницы «Формування документу»;
- данные из «Налаштування → Формування документу» (по scope текущей организации).

Поддерживаются синхронный и асинхронный режимы генерации.

## Актуальный пользовательский сценарий

1. После сохранения операции открывается модал «Сформувати документи?».
2. В модале показываются документы операции (`doc_name`) и переход на билдера.
3. На странице `/operation-documents-builder` пользователь:
    - выбирает, какие документы формировать;
    - задаёт ФИО/состав комиссии/ответственного (или оставляет дефолты);
    - нажимает «Сформувати документи».
4. Для каждого выбранного документа выполняется генерация и скачивание DOCX.

## Источники данных и приоритет

### 1) Роли/персоны

Ключи: `receiver_name`, `commission_head`, `commission_member_1..3`, `responsible_person`.

Приоритет:

1. Значение, выбранное пользователем в билдере (`/operation-documents-builder`).
2. Если не выбрано — дефолт из настроек формирования документов текущей организации.
3. Если всё пусто — `-`.

### 2) Поля организации

Ключи: `edrpou`, `document_year`, `commission_order_details`.

Источник:

- берутся из настроек формирования документов текущей организации;
- для `document_year`, если пусто, берётся текущий год;
- если после этого пусто — `-`.

### 3) Данные операции

Берутся из самой операции, её документов и товаров/позиций (`products` или `draft_products`).

## API

### `GET /api/operation-doc-templates`

Возвращает доступные шаблоны и автоматически подобранный `resolved_template_id`.

### `POST /api/operations/{id}/generate-doc`

Формирует документ по операции.

Тело запроса:

- `action`: `preview` | `download`
- `template_id`: `default` или `file:<filename.docx>`
- `document_name`: имя конкретного документа операции (для точной подстановки `doc_*`)
- `settings`: объект с полями билдера (роли/персоны)

Ответ:

- sync: `download_url` (и `preview_url` для preview при успешной конвертации)
- async: `queued=true`, `task_id`

### `GET /api/operations/doc-generation-status/{taskId}`

Проверка статуса асинхронной задачи генерации.

### `GET /generated-documents/{token}`

Скачивание/открытие ранее сгенерированного файла по токену.

## Шаблоны

### Виды шаблонов

- Встроенный: `default`.
- Пользовательские: `storage/app/templates/operations/*.docx`.

### Автоподбор шаблона

Система сравнивает `doc_name` документа операции с именем файла шаблона (без учёта регистра и служебных символов).

Пример:

- документ: `Акт списання запасів`
- шаблон: `storage/app/templates/operations/Акт списання запасів.docx`

Если совпадение не найдено, используется `default`.

## Каталог плейсхолдеров

Ниже перечислены плейсхолдеры, которые реально подставляются в пользовательские DOCX.

### Операция

- `${operation_number}` — ID операции.
- `${operation_date}` — дата операции в формате `dd.mm.YYYY`.
- `${operation_type}` — название типа операции.
- `${counteragent}` — контрагент операции.

### Выбранный документ операции (`doc_*`)

- `${doc_name}` — название выбранного документа операции.
- `${doc_num}` — номер выбранного документа.
- `${doc_date}` — дата выбранного документа.
- `${doc_count}` — количество документов в операции.

Примечание: `selected document` определяется так:

1. По `document_name`, переданному из билдера.
2. Если не найден — по совпадению с именем шаблона.
3. Иначе — первый документ операции.

### Настройки формирования (персоны)

- `${receiver_name}` — «На кого документ» (только ПІБ).
- `${receiver_position}` — должность получателя.
- `${receiver_full}` — получатель в формате `должность + ПІБ`.
- `${receiver}` — алиас `${receiver_name}`.
- `${doc_receiver}` — алиас `${receiver_name}`.
- `${commission_head}` — голова комиссии в формате `должность + ПІБ`.
- `${commission_head_name}` — только ПІБ головы комиссии.
- `${commission_head_position}` — только должность головы комиссии.
- `${commission_member_1}` — член комиссии №1 в формате `должность + ПІБ`.
- `${commission_member_1_name}` — только ПІБ члена комиссии №1.
- `${commission_member_1_position}` — только должность члена комиссии №1.
- `${commission_member_2}` — член комиссии №2 в формате `должность + ПІБ`.
- `${commission_member_2_name}` — только ПІБ члена комиссии №2.
- `${commission_member_2_position}` — только должность члена комиссии №2.
- `${commission_member_3}` — член комиссии №3 в формате `должность + ПІБ`.
- `${commission_member_3_name}` — только ПІБ члена комиссии №3.
- `${commission_member_3_position}` — только должность члена комиссии №3.
- `${commission_members}` — строка из членов комиссии 1–3 в формате `должность + ПІБ`, через запятую.
- `${responsible_person}` — материально-ответственное лицо (только ПІБ).
- `${responsible_person_name}` — только ПІБ материально-ответственного.
- `${responsible_person_position}` — только должность материально-ответственного.
- `${responsible_person_full}` — материально-ответственный в формате `должность + ПІБ`.
- `${materially_responsible_person}` — алиас `${responsible_person}`.
- `${materially_responsible_person_name}` — алиас `${responsible_person_name}`.
- `${materially_responsible_person_position}` — алиас `${responsible_person_position}`.
- `${materially_responsible_person_full}` — алиас `${responsible_person_full}`.

### Настройки формирования (реквизиты организации)

- `${edrpou}` — ЄДРПОУ организации.
- `${organization_edrpou}` — алиас `${edrpou}`.
- `${org_edrpou}` — алиас `${edrpou}`.
- `${document_year}` — год документа.
- `${doc_year}` — алиас `${document_year}`.
- `${year}` — алиас `${document_year}`.
- `${commission_order_details}` — дата и № приказа о назначении комиссии.
- `${commission_order}` — алиас `${commission_order_details}`.
- `${commission_order_info}` — алиас `${commission_order_details}`.

### Строки документов (clone row)

Для повторяющихся строк документов:

- `${doc_row_no}` — номер строки документа (1, 2, 3...).
- `${doc_row_name}` — название документа.
- `${doc_row_num}` — номер документа.
- `${doc_row_date}` — дата документа.

### Строки товаров (clone row)

Для повторяющихся строк товаров:

- `${row_no}` — номер строки товара.
- `${row_name}` — наименование товара.
- `${row_quantity}` — количество.
- `${row_unit}` — единица измерения.
- `${row_price}` — цена.
- `${row_sum}` — сумма.
- `${row_inv_number}` — инвентарный номер.

### Итоги

- `${total_quantity}` — итоговое количество.
- `${total_sum}` — итоговая сумма.
- `${operation_total_quantity}` — алиас `${total_quantity}`.
- `${operation_total_sum}` — алиас `${total_sum}`.
- `${total_quantity_text}` — итоговое количество прописью (укр).
- `${total_sum_text}` — итоговая сумма прописью (укр, грн/коп).

## Что важно учесть в DOCX-шаблонах

- Используйте плейсхолдеры строго в формате `${name}`.
- Для таблиц используйте clone-row плейсхолдеры (`doc_row_*`, `row_*`) в одной строке-шаблоне.
- Если данных нет, генератор подставляет `-` (для большинства текстовых полей).

## Аудит

События генерации пишутся в таблицу `samvol_inventory_document_generations`

## Превью (PDF)

- Для `action=preview` система пытается конвертировать DOCX в PDF.
- Для конвертации нужен LibreOffice (`soffice`) в headless-режиме.
- Путь можно задать через `.env`: `DOC_PREVIEW_SOFFICE_PATH`.
